# 監査ログと計画進捗管理

このファイルは、ワークフロー全体を通じて一貫した監査証跡を残すためのルール集。セッション中断からの再開、判断の追跡可能性、レビュワーによる事後検証を支えるための設計。

---

## 監査ログ（audit.md）

### 記録対象

すべてのユーザー入力を、タイムスタンプ付きで `aidlc-docs/audit.md` に記録する。ログの対象は以下:

- プロンプト、質問、応答のすべて
- 承認プロンプト（ユーザーに尋ねる前にも記録する）
- ユーザー応答（受け取った直後に記録する）

### 書式

タイムスタンプは ISO 8601 形式（`YYYY-MM-DDTHH:MM:SSZ`）。各エントリにステージのコンテキストを含める。

```markdown
## [ステージ名またはインタラクションの種類]

**タイムスタンプ**: [ISO タイムスタンプ]
**ユーザー入力**: "[完全な生のユーザー入力 — 要約禁止]"
**AI 応答**: "[AI の応答または実行されたアクション]"
**コンテキスト**: [ステージ、アクション、または下した決定]

---
```

### 生の入力をそのまま記録する理由

要約すると、後から「ユーザーが本当は何を要求したか」を復元できなくなる。特にセッションが中断して別セッションで再開するとき、原文がないと意図の取り違えが発生する。

- ユーザー入力は提供されたとおりに正確にキャプチャする（要約・言い換えしない）
- 承認だけでなく、すべてのインタラクションを記録する

### 追記 vs 上書き

`audit.md` は必ず追記する。既存内容を上書きするツールやコマンドは使わない。上書きすると、これまでの監査証跡が失われ、二度と復元できない。

**正しい方法**:

1. `audit.md` ファイルを読み込む
2. Edit で末尾に追記するか、Read → 末尾追加 → Write の順で行う

**誤った方法**:

1. `audit.md` ファイルを読み込む
2. 読み込んだ内容を丸ごとコピーし、末尾に新しい変更を足して Write で上書きする（これは一見同じ結果に見えるが、差分レビューが困難になり、途中で編集が混入した場合に検出できない）

## 計画レベルのチェックボックス強制適用

### ルール

- 計画ファイルに記載されたステップを完了したら、その直後に `[x]` とマークする
- マークは作業が完了した同じインタラクション内で行う（あとでまとめて更新しない — セッション中断時に進捗が欠落するため）
- 計画チェックボックスを更新しないまま作業を完了させない

### 2 レベルのチェックボックストラッキング

- **計画レベル**: 各ステージ内の詳細な実行進捗を追跡する
- **ステージレベル**: `aidlc-docs/aidlc-state.md` でワークフロー全体の進捗を追跡する

進捗更新は作業が完了した同じインタラクションで行う。こうすることで、セッションが中断しても次回の再開時に完了済み作業とこれから行う作業が一目で分かる。
