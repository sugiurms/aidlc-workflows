# セッション継続テンプレート

## ウェルカムバックプロンプトのテンプレート
ユーザーが既存の AI-DLC プロジェクトの続きに戻ってきた際は、このプロンプトを提示してください:

```markdown
**おかえりなさい！進行中の AI-DLC プロジェクトがあることを確認しました。**

aidlc-state.md によると、現在の状況は以下の通りです:
- **プロジェクト**: [プロジェクト名]
- **現在のフェーズ**: [INCEPTION/CONSTRUCTION/OPERATIONS]
- **現在のステージ**: [ステージ名]
- **最後に完了した内容**: [最後に完了したステップ]
- **次のステップ**: [次に取り組む内容]

**本日は何に取り組みますか？**

A) 前回の続きから再開する（[次のステップの説明]）
B) 前のステージを確認する（[利用可能なステージを表示]）

[Answer]: 
```

## 必須: セッション継続の手順
1. **既存プロジェクトを検出した際は常に aidlc-state.md を最初に読み込む**
2. ワークフローファイルから**現在のステータスを解析**してプロンプトに反映する
3. **必須: 前のステージのアーティファクトを読み込む** — ステージを再開する前に、前のステージの関連アーティファクトをすべて自動的に読み込む:
   - **リバースエンジニアリング**: architecture.md、code-structure.md、api-documentation.md を読み込む
   - **要件分析**: requirements.md、requirement-verification-questions.md を読み込む
   - **ユーザーストーリー**: stories.md、personas.md、story-generation-plan.md を読み込む
   - **アプリケーション設計**: アプリケーション設計アーティファクト（components.md、component-methods.md、services.md）を読み込む
   - **設計（ユニット）**: unit-of-work.md、unit-of-work-dependency.md、unit-of-work-story-map.md を読み込む
   - **ユニットごとの設計**: functional-design.md、nfr-requirements.md、nfr-design.md、infrastructure-design.md を読み込む
   - **コードステージ**: すべてのコードファイル、計画、および前のすべてのアーティファクトを読み込む
4. **ステージ別のスマートなコンテキスト読み込み**:
   - **初期ステージ（ワークスペース検出、リバースエンジニアリング）**: ワークスペース分析を読み込む
   - **要件/ストーリー**: リバースエンジニアリング + 要件のアーティファクトを読み込む
   - **設計ステージ**: 要件 + ストーリー + アーキテクチャ + 設計のアーティファクトを読み込む
   - **コードステージ**: すべてのアーティファクト + 既存のコードファイルを読み込む
5. アーキテクチャの選択と現在のフェーズに基づいて**選択肢を適応**させる
6. 一般的な説明ではなく**具体的な次のステップを表示**する
7. タイムスタンプ付きで継続プロンプトを **audit.md に記録**する
8. **コンテキストサマリー**: アーティファクトを読み込んだ後、ユーザーの認識向上のために読み込んだ内容の簡潔なサマリーを提供する
9. **質問の行い方**: 確認またはユーザーフィードバックの質問は常に `.md` ファイルに配置して行うこと。多肢選択問題をチャットセッション内にインラインで配置してはいけません。

## エラー処理
セッション再開中にアーティファクトが欠けているまたは破損している場合は、回復手順について [error-handling.md](error-handling.md) を参照してください。
