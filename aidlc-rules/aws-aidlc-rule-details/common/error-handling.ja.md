# エラー処理と回復手順

## 一般的なエラー処理の原則

### エラーが発生した場合
1. **エラーを特定する**: 何が問題だったかを明確に述べる
2. **影響を評価する**: エラーがブロッキングであるか、回避策で対処できるかを判断する
3. **通知する**: エラーと選択肢についてユーザーに知らせる
4. **解決策を提示する**: 解決または回避するための明確な手順を提供する
5. **記録する**: `audit.md` にエラーと解決策を記録する

### エラー重大度レベル

**致命的**: ワークフローを続行できない
- 必要なファイルまたはアーティファクトが見つからない
- 処理できない無効なユーザー入力
- ファイル操作を妨げるシステムエラー

**高**: ステージが計画通り完了できない
- 必要な質問への回答が不完全
- ユーザーの矛盾した回答
- 前のステージからの依存関係の欠如

**中**: 回避策でステージを継続できる
- オプションのアーティファクトが見つからない
- 重要でない検証の失敗
- 部分的な完了が可能

**低**: 進行を妨げないマイナーな問題
- フォーマットの不整合
- オプション情報の欠如
- 非ブロッキングの警告

## ステージ別エラー処理

### ワークスペース検出のエラー

**エラー**: ワークスペースファイルを読み込めない
- **原因**: 権限の問題、ディレクトリの欠如
- **解決策**: ユーザーにワークスペースパスと権限の確認を依頼する
- **回避策**: ユーザー提供の情報のみで進行する

**エラー**: 既存の `aidlc-state.md` が破損している
- **原因**: 手動編集、前回の実行の不完全な終了
- **解決策**: ユーザーに一から始めるか回復を試みるかを尋ねる
- **回復**: バックアップを作成し、新しい状態ファイルを始める

**エラー**: 必要なステージを決定できない
- **原因**: ユーザーからの情報が不十分
- **解決策**: 意図とスコープについて確認のための質問をする
- **回避策**: 包括的な実行計画をデフォルトとする

### 要件分析のエラー

**エラー**: ユーザーが矛盾した要件を提供する
- **原因**: 理解が不明確、ニーズの変化
- **解決策**: 矛盾を解消するためのフォローアップ質問を作成する
- **続行禁止**: 矛盾が解消されるまで

**エラー**: 要件ドキュメントを変換できない
- **原因**: サポートされていない形式、ファイルの破損
- **解決策**: サポートされている形式で要件を提供するようユーザーに依頼する
- **回避策**: ユーザーの口頭による説明で作業する

**エラー**: 確認の質問への回答が不完全
- **原因**: ユーザーが質問をスキップした、何を答えればよいか不明
- **解決策**: 未回答の質問を強調し、例を提供する
- **続行禁止**: すべての必須質問が回答されるまで

### ユーザーストーリーのエラー

**エラー**: 要件をストーリーにマップできない
- **原因**: 要件が曖昧すぎる、機能的な詳細が不足
- **解決策**: 要件分析に戻って明確化する
- **回避策**: 利用可能な情報に基づいてストーリーを作成し、未完了としてマークする

**エラー**: ユーザーがストーリー計画の回答で曖昧な表現をする
- **原因**: 不明確な選択肢、複雑な判断
- **解決策**: 具体的な例を含むフォローアップ質問を追加する
- **続行禁止**: 曖昧さが解消されるまで

**エラー**: ストーリー生成計画に未完了のステップがある
- **原因**: 実行が中断された、ステップがスキップされた
- **解決策**: 最初の未完了ステップから再開する
- **回復**: 完了したステップを確認し、チェックポイントから継続する

### アプリケーション設計のエラー

**エラー**: アーキテクチャの決定が不明確または矛盾している
- **原因**: 曖昧な回答、要件の競合
- **解決策**: 決定を明確化するためのフォローアップ質問を追加する
- **続行禁止**: 決定が明確になり文書化されるまで

**エラー**: サービス/ユニットの数を決定できない
- **原因**: 境界に関する情報が不足
- **解決策**: デプロイメント、チーム構造、スケーリングに関する具体的な質問をする
- **回避策**: モノリスをデフォルトとし、後で変更を許可する

### 設計のエラー

**エラー**: ユニットの依存関係が循環している
- **原因**: 境界定義の不備、密結合
- **解決策**: 循環依存を特定し、リファクタリングを提案する
- **回復**: 循環を断ち切るためにユニット境界を修正する

**エラー**: ユニット設計計画にステップが欠けている
- **原因**: 計画生成が不完全、テンプレートエラー
- **解決策**: 必要なすべてのステップで計画を再生成する
- **回復**: 欠けているステップを既存の計画に追加する

**エラー**: 設計アーティファクトを生成できない
- **原因**: ユニット情報の欠如、要件が不明確
- **解決策**: ユニット定義を明確化するためにユニット計画に戻る
- **回避策**: 部分的な設計を生成し、ギャップをマークする

### 非機能要件実装のエラー

**エラー**: 技術スタックの選択に互換性がない
- **原因**: 要件の競合、プラットフォームの制限
- **解決策**: 非互換性を強調し、ユーザーに選択を促す
- **続行禁止**: 互換性のある選択がなされるまで

**エラー**: 組織上の制約を満たせない
- **原因**: ネットワーク制限、セキュリティポリシー
- **解決策**: 制約を文書化し、ユーザーに回避策を尋ねる
- **エスカレーション**: 設定に人間の介入が必要な場合がある

**エラー**: 非機能要件実装ステップに人間の対応が必要
- **原因**: AI が特定のタスクを実行できない（ネットワーク設定、資格情報）
- **解決策**: **人間のタスク (HUMAN TASK)** としてはっきりマークし、手順を提供する
- **待機**: 続行前にユーザーの確認を待つ

### コード生成計画のエラー

**エラー**: コード生成計画が不完全
- **原因**: 設計アーティファクトの欠如、要件が不明確
- **解決策**: アーティファクトを完成させるために設計ステージに戻る
- **回復**: 利用可能な情報で計画を生成し、ギャップをマークする

**エラー**: ユニットの依存関係が満たされていない
- **原因**: 依存するユニットがまだ生成されていない
- **解決策**: 依存関係を考慮して生成順序を並べ替える
- **回避策**: スタブな依存関係で生成し、後で統合する

### コード生成のエラー（パート 2: コード生成）

**エラー**: ステップのコードを生成できない
- **原因**: 設計情報の不足、要件が不明確
- **解決策**: ステップをスキップし、未完了として記録して継続する
- **回復**: 追加情報を収集した後にステップに戻る

**エラー**: 生成されたコードに構文エラーがある
- **原因**: テンプレートの問題、言語固有の問題
- **解決策**: 構文エラーを修正し、必要に応じて再生成する
- **検証**: 続行前にコードがコンパイルされることを確認する

**エラー**: テスト生成が失敗する
- **原因**: 複雑なロジック、テストフレームワークの設定が未完成
- **解決策**: 基本的なテスト構造を生成し、手動完了のためにマークする
- **回避策**: テストなしで進行し、オペレーションフェーズで追加する

### オペレーションのエラー

**エラー**: ビルドツールを決定できない
- **原因**: 通常とは異なるプロジェクト構造、複数のビルドシステム
- **解決策**: ビルドツールとコマンドを指定するようユーザーに依頼する
- **回避策**: 一般的な手順を提供し、ユーザーが適応する

**エラー**: デプロイ先が不明確
- **原因**: 複数の環境、複雑なインフラ
- **解決策**: デプロイ先と方法を指定するようユーザーに依頼する
- **回避策**: 一般的なプラットフォーム向け手順を提供する

## 回復手順

### ステージの部分的な完了

**シナリオ**: ステージが実行中に中断された

**回復ステップ**:
1. ステージ計画ファイルを読み込む
2. 最後に完了したステップ（最後の [x] チェックボックス）を特定する
3. 次の未完了ステップから再開する
4. 直前のすべてのステップが実際に完了していることを確認する
5. 通常の実行を継続する

### 状態ファイルの破損

**シナリオ**: `aidlc-state.md` が破損または不整合

**回復ステップ**:
1. バックアップを作成する: `aidlc-state.md.backup`
2. ユーザーに現在実際にどのステージにいるか尋ねる
3. 状態ファイルをゼロから再生成する
4. 既存のアーティファクトに基づいて完了済みステージをマークする
5. 現在のステージから再開する

### アーティファクトの欠如

**シナリオ**: 前のステージからの必要なアーティファクトが見つからない

**回復ステップ**:
1. どのアーティファクトが欠けているかを特定する
2. 再生成できるかどうかを確認する
3. 可能な場合: そのステージに戻ってアーティファクトを再生成する
4. 不可能な場合: ユーザーに手動で情報を提供するよう依頼する
5. `audit.md` にギャップを記録する

### ユーザーがステージの再実行を希望する

**シナリオ**: ユーザーがステージの結果に不満でやり直したい

**回復ステップ**:
1. 再実行したいことを確認する（データが失われる）
2. 既存のアーティファクトをアーカイブする: `{artifact}.backup`
3. `aidlc-state.md` のステージステータスをリセットする
4. 計画ファイルのステージチェックボックスをクリアする
5. ステージを最初から再実行する

### ユーザーがステージのスキップを希望する

**シナリオ**: ユーザーが計画されていたステージをスキップしたい

**回復ステップ**:
1. 影響について理解していることを確認する
2. `audit.md` にスキップの理由を記録する
3. `aidlc-state.md` でステージを「SKIPPED」としてマークする
4. 次のステージに進む
5. 注記: 依存関係が欠如している場合、後のステージで問題が発生する可能性がある

## エスカレーションの指針

### ユーザーの助けを求めるタイミング

**即時**:
- 矛盾または曖昧なユーザー入力
- 必要な情報の欠如
- AI が解決できない技術的制約
- ビジネス判断が必要な決定

**解決を試みた後**:
- 同じステップでのエラーの繰り返し
- 複雑な技術的問題
- 特殊なプロジェクト構造
- 外部システムとの統合

### やり直しを提案するタイミング

**やり直しを検討する場合**:
- 複数のステージにエラーがある
- 状態ファイルが深刻に破損している
- ユーザーの要件が大幅に変更された
- アーキテクチャの決定を覆す必要がある
- ユーザーが欠けている情報を提供できない
- アーティファクトがフェーズ間で不整合

**やり直し前に**:
1. すべての既存の作業をアーカイブする
2. 学んだことを記録する
3. 保持するものを特定する
4. ユーザーの確認を得る
5. 新しい実行計画を作成する

## セッション再開時のエラー

### 再開時のアーティファクト欠如

**エラー**: 前のステージからの必要なアーティファクトが見つからない
- **原因**: ファイルが削除された、移動された、または作成されなかった
- **解決策**:
  1. どのステージが欠けているアーティファクトを作成したかを特定する
  2. aidlc-state.md でステージが完了としてマークされているか確認する
  3. 完了としてマークされているがアーティファクトが欠けている場合: そのステージを再生成する
  4. 完了としてマークされていない場合: そのステージから再開する
- **回復**: 欠けているアーティファクトを作成するステージに戻って再実行する

**エラー**: アーティファクトファイルは存在するが空または破損している
- **原因**: 書き込みの中断、手動編集、ファイルシステムの問題
- **解決策**:
  1. 破損したファイルのバックアップを作成する
  2. 作成したステージから再生成を試みる
  3. 再生成できない場合: 情報を再作成するためにユーザーに依頼する
- **回復**: アーティファクトを作成するステージを再実行する

### 再開時の状態の不整合

**エラー**: aidlc-state.md ではステージが完了しているが、アーティファクトが存在しない
- **原因**: 状態ファイルは更新されたがアーティファクト生成が失敗した
- **解決策**:
  1. aidlc-state.md でステージを未完了としてマークする
  2. アーティファクトを生成するためにステージを再実行する
  3. 完了とマークする前にアーティファクトが存在することを確認する
- **回復**: ステージのステータスをリセットして再実行する

**エラー**: アーティファクトは存在するが aidlc-state.md ではステージが未完了
- **原因**: アーティファクト生成は成功したが状態の更新が失敗した
- **解決策**:
  1. アーティファクトが完全で有効であることを確認する
  2. aidlc-state.md を更新してステージを完了としてマークする
  3. 次のステージに進む
- **回復**: 状態ファイルを更新して実際の完了状態を反映する

**エラー**: aidlc-state.md で複数のステージが「current」としてマークされている
- **原因**: 状態ファイルの破損、手動編集
- **解決策**:
  1. アーティファクトを確認して実際の進捗状況を判断する
  2. ユーザーに現在実際にどのステージにいるか尋ねる
  3. aidlc-state.md を修正して単一の現在のステージを表示する
- **回復**: 既存のアーティファクトに基づいて状態ファイルを再構築する

### コンテキスト読み込みのエラー

**エラー**: 前のステージから必要なコンテキストを読み込めない
- **原因**: ファイルの欠如、コンテンツの破損、ファイルパスの誤り
- **解決策**:
  1. 現在のステージに必要なアーティファクトをリストする
  2. 欠けているまたは破損しているものを確認する
  3. 欠けているアーティファクトを再生成するか、ユーザーに情報を依頼する
- **回復**: 現在のステージを再開する前に前提条件のステージを完了する

**エラー**: 読み込んだアーティファクトに矛盾する情報が含まれている
- **原因**: 手動編集、複数人での作業、不完全な更新
- **解決策**:
  1. 矛盾を特定してユーザーに提示する
  2. どの情報が正しいかをユーザーに尋ねる
  3. 矛盾を解消するためにアーティファクトを更新する
- **回復**: 進行前に矛盾を解消する

### 再開時のベストプラクティス

1. **常に状態を検証する**: aidlc-state.md が実際のアーティファクトと一致していることを確認する
2. **段階的に読み込む**: アーティファクトをステージごとに読み込み、それぞれを検証する
3. **早期に失敗する**: 重要なアーティファクトが欠けている場合は即座に停止する
4. **明確に伝える**: 何が欠けているか、なぜ必要なのかをユーザーに正確に伝える
5. **選択肢を提示する**: 再生成、手動提供、またはやり直し
6. **回復を記録する**: すべての回復アクションを audit.md に記録する

## ログ記録の要件

### エラーログのフォーマット

```markdown
## Error - [ステージ名]
**Timestamp**: [ISO タイムスタンプ]
**Error Type**: [Critical/High/Medium/Low]
**Description**: [何が問題だったか]
**Cause**: [なぜ発生したか]
**Resolution**: [どのように解決したか]
**Impact**: [ワークフローへの影響]

---
```

### 回復ログのフォーマット

```markdown
## Recovery - [ステージ名]
**Timestamp**: [ISO タイムスタンプ]
**Issue**: [何が回復を必要としたか]
**Recovery Steps**: [何が行われたか]
**Outcome**: [回復の結果]
**Artifacts Affected**: [影響を受けたファイルのリスト]

---
```

## 予防のベストプラクティス

1. **早期に検証する**: 作業を開始する前に入力と依存関係を確認する
2. **頻繁にチェックポイントを取る**: ステップ完了後すぐにチェックボックスを更新する
3. **明確に伝える**: 何をしているか、なぜしているかを説明する
4. **質問する**: 曖昧さを即座に明確化する — 仮定しない
5. **すべてを記録する**: すべての決定と変更を `audit.md` に記録する
