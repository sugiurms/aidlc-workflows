# 👋 AI-DLC（AI 主導開発ライフサイクル）へようこそ！ 👋

あなたの特定のニーズに合わせてインテリジェントに調整される、適応型ソフトウェア開発ワークフローをご案内します。

## AI-DLC とは？

AI-DLC は、プロジェクトのニーズに適応する構造化かつ柔軟なソフトウェア開発プロセスです。経験豊富なソフトウェアアーキテクトが以下のことを行ってくれると考えてください:

- **要件を分析し**、必要に応じて確認の質問をする
- **複雑さとリスクに基づいて最適なアプローチを計画**する
- 単純な変更には**不要なステップをスキップ**しながら、複雑なプロジェクトには包括的な対応を行う
- **すべてを文書化**することで、決定事項と根拠の完全な記録を残す
- 明確なチェックポイントと承認ゲートを持つ**各フェーズを通じて案内**する

### フェーズの概要:

**インセプションフェーズ (INCEPTION PHASE)** - *計画とアプリケーション設計*
- **目的**: 何を構築するか、なぜ構築するかを決定する
- **活動内容**: 要件の理解、既存コードの分析（ある場合）、アプローチの計画
- **成果物**: 明確な要件、実行計画、並行開発のための作業単位数の決定
- **あなたの役割**: 質問に回答し、計画をレビューし、方向性を承認する

**コンストラクションフェーズ (CONSTRUCTION PHASE)** - *詳細な設計、実装・テスト*
- **目的**: どのように構築するかを決定する
- **活動内容**: 詳細な設計（必要な場合）、コード生成、包括的なテスト
- **成果物**: 動作するコード、テスト、ビルド手順
- **あなたの役割**: 設計をレビューし、実装計画を承認し、結果を検証する

**オペレーションフェーズ (OPERATIONS PHASE)** - *デプロイとモニタリング（将来）*
- **目的**: どのようにデプロイして運用するか
- **ステータス**: 将来のデプロイとモニタリングワークフローのプレースホルダー
- **現在の状況**: ビルドとテストの活動はコンストラクションフェーズで処理される

## 主要な原則:

- ⚡ **完全に適応的**: 各ステージがあなたのニーズに基づいて独立して評価される
- 🎯 **効率的**: 単純な変更には必要不可欠なステージのみ実行する
- 📋 **包括的**: 複雑な変更にはすべての安全策を含む完全な処理を行う
- 🔍 **透明性**: 作業開始前に実行計画を確認・承認できる
- 📝 **文書化**: すべての決定と変更の完全な監査証跡を保持する
- 🎛️ **ユーザーコントロール**: ステージの追加または除外をリクエストできる

## 次に起こること:

1. **ワークスペースを分析**して、これが新規プロジェクトか既存プロジェクトかを把握する
2. **要件を収集**し、必要な場合は確認の質問をする
3. **実行計画を作成**して、実行するステージとその理由を提示する
4. **計画をレビューして承認**する（または変更をリクエストする）
5. 各主要ステージにチェックポイントを設けながら**計画を実行**する
6. 完全なドキュメントとテストを伴う**動作するコードを受け取る**

AI-DLC プロセスは以下に適応します:
- 📋 あなたの意図の明確さと複雑さ
- 🔍 既存のコードベースの状態
- 🎯 変更のスコープと影響
- ⚡ リスクと品質要件

始めましょう！
