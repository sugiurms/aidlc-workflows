# コンテンツ検証ルール

## 必須: ファイル作成前のコンテンツ検証

**重要**: 生成されたすべてのコンテンツは、ファイルへの書き込み前に検証してパースエラーを防ぐ必要があります。

## ASCII 図版標準

**重要**: ASCII 図版を含むファイルを作成する前に:

1. `common/ascii-diagram-standards.md` を**読み込む**
2. 各図版を**検証する**:
   - 1 行ごとの文字数を数える（すべての行が同じ幅でなければならない）
   - 使用できる文字: `+` `-` `|` `^` `v` `<` `>` とスペースのみ
   - Unicode ボックス描画文字は使用不可
   - スペースのみ使用（タブ不可）
3. ボックスのコーナーが垂直に揃っているか確認して整列を**テスト**する

**パターンおよび検証チェックリストについては `common/ascii-diagram-standards.md` を参照してください。**

## Mermaid 図版の検証

### 必要な検証手順
1. **構文チェック**: ファイル作成前に Mermaid 構文を検証する
2. **文字エスケープ**: 特殊文字が適切にエスケープされていることを確認する
3. **フォールバックコンテンツ**: Mermaid が検証に失敗した場合はテキスト代替を提供する

### Mermaid 検証ルール
```markdown
## Mermaid 図版を含むファイルを作成する前に:

1. ノード ID に無効な文字がないか確認する（英数字とアンダースコアのみ使用可）
2. ラベル内の特殊文字をエスケープする: " → \" および ' → \'
3. フローチャートの構文を検証する: ノード間の接続が有効であること
4. 簡単な検証でダイアグラムのパースをテストする

## フォールバック: Mermaid の検証が失敗した場合、テキストベースのワークフロー表現を使用する
```

### 実装パターン
```markdown
## ワークフロー可視化

### Mermaid 図版 (構文が有効な場合)
```mermaid
[検証済みの図版内容]
```

### テキスト代替 (常に含めること)
```
Phase 1: INCEPTION
- Stage 1: Workspace Detection (COMPLETED)
- Stage 2: Requirements Analysis (COMPLETED)
[テキスト表現を続ける]
```

## 一般コンテンツ検証

### 作成前の検証チェックリスト
- [ ] 埋め込みコードブロック (Mermaid、JSON、YAML) を検証する
- [ ] 特殊文字のエスケープを確認する
- [ ] Markdown 構文の正確性を検証する
- [ ] コンテンツのパース互換性をテストする
- [ ] 複雑な要素のフォールバックコンテンツを含める

### エラー防止ルール
1. **ファイル書き込みツール/コマンドを使用する前に必ず検証する**: 未検証のコンテンツは書き込まない
2. **特殊文字をエスケープする**: 特に図版やコードブロック内で
3. **代替手段を提供する**: ビジュアルコンテンツのテキスト版を含める
4. **構文をテストする**: 複雑なコンテンツ構造を検証する

## 検証失敗時の処理

### 検証が失敗した場合
1. **エラーを記録する**: 検証に失敗した内容を記録する
2. **フォールバックコンテンツを使用する**: テキストベースの代替に切り替える
3. **ワークフローを継続する**: コンテンツ検証の失敗でブロックしない
4. **ユーザーに通知する**: パース制約のため簡略化されたコンテンツを使用したことを伝える
